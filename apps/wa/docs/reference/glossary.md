# Work Allocation Glossary

| Term | Definition |
|------|------------|
| **ABAC** | Attribute-Based Access Control. The model used by `wa-task-management-api` to gate task access: a user's AM role assignments are evaluated against per-task `task_roles` rows, matching on case ID, role name, and required permission flags. See [Access Control](../explanation/access-control.md). |
| **additionalData** | The `AdditionalData` field in a CCD case-event message. Contains `Data` (case field values) and `Definition` (CCD type metadata). DMN inputs access fields via FEEL expressions like `additionalData.Data.appealType`. See [Case Event Handler](../explanation/case-event-handler.md). |
| **allowedJurisdictions** | The `config.allowedJurisdictions` setting on `wa-task-management-api` that gates which jurisdictions' events are processed. Currently: `ia`, `wa`, `sscs`, `civil`, `publiclaw`, `privatelaw`, `employment`, `st_cic`. See [Overview](../explanation/overview.md). |
| **am-role-assignment-service** | The upstream Access Management service that stores and serves role assignments. `wa-task-management-api` calls it to validate every user-facing task action. See [Access Control](../explanation/access-control.md). |
| **ASB** | Azure Service Bus. The messaging backbone that CCD uses to publish case-event messages to WA. `wa-case-event-handler` subscribes to the `ccd-case-events` topic via a session-based subscription. See [Architecture](../explanation/architecture.md). |
| **autoAssignable** | A flag on a Permissions DMN row (and the resulting `task_roles` record) that allows the auto-assignment query to consider a role for automatic assignment at task creation time. Only roles with `Own` permission and `autoAssignable=true` are candidates. See [Task Lifecycle](../explanation/task-lifecycle.md). |
| **BPMN** | Business Process Model and Notation. WA uses a single generic Camunda BPMN process (`wa-task-initiation-ia-asylum`) to manage task lifecycle across all jurisdictions. See [BPMN Workflows](../explanation/bpmn-workflows.md). |
| **Camunda** | The open-source BPMN/DMN workflow engine used by WA. WA's services communicate with Camunda via the `wa-workflow-api` proxy. See [Architecture](../explanation/architecture.md). |
| **cancelTasks** | The Camunda BPMN message name used to cancel an active task process instance. Sent by `wa-case-event-handler` `CancellationCaseEventHandler` when a CCD event matches a `Cancel` rule in the cancellation DMN. See [BPMN Workflows](../explanation/bpmn-workflows.md). |
| **canReconfigure** | A boolean output column in the Configuration DMN. When `true`, the row is re-applied during reconfiguration events; when `false` or absent, the attribute is set only once at initiation. See [DMN Task Configuration](../explanation/dmn-task-configuration.md) and [DMN Schema](dmn-schema.md). |
| **caseworker-wa-task-configuration** | The CCD role that must be granted read access to all case fields referenced in a service team's Configuration DMN. Without this, task configuration will fail silently (returning null for case data fields). See [How-to: Onboard Jurisdiction](../how-to/onboard-jurisdiction.md). |
| **ccd-case-events** | The Azure Service Bus topic that CCD publishes case event messages to. `wa-case-event-handler` subscribes with a session-based subscription (`wa-ccd-case-events-sub-{env}`). See [Case Event Handler](../explanation/case-event-handler.md). |
| **CEH** | Short form for `wa-case-event-handler`. The service that bridges ASB CCD events to the WA task pipeline. See [Case Event Handler](../explanation/case-event-handler.md). |
| **cft_task_db** | The authoritative PostgreSQL database for WA tasks, owned by `wa-task-management-api`. Core tables: `tasks`, `task_roles`, `work_types`, `sensitive_task_event_logs`. See [Architecture](../explanation/architecture.md). |
| **CFTTaskState** | The Java enum (`CFTTaskState.java`) defining all task states stored in `cft_task_db`. States: `UNCONFIGURED`, `PENDING_AUTO_ASSIGN`, `CONFIGURED`, `UNASSIGNED`, `ASSIGNED`, `PENDING_RECONFIGURATION`, `COMPLETED`, `CANCELLED`, `TERMINATED`. See [Task States](task-states.md). |
| **COLLECT** | A Camunda DMN hit policy where all matching rules fire and all their outputs are returned. Used in the initiation, cancellation, completion, and task-types DMN tables. See [DMN Schema](dmn-schema.md). |
| **Configuration DMN** | The DMN table (`wa-task-configuration-{jurisdiction}-{caseType}`) that enriches task attributes from CCD case data after a task is initiated. Hit policy: RULE ORDER. Outputs key-value pairs consumed by the date calculation engine. See [DMN Task Configuration](../explanation/dmn-task-configuration.md). |
| **CRA** | Cross-Role Assignment. Allows users outside a task's "natural" role category to complete it. Governed by the `OWN` vs `EXECUTE` permission distinction on `task_roles` rows. See [Access Control](../explanation/access-control.md). |
| **createTaskMessage** | The Camunda BPMN message name that starts a new standalone task process instance. Sent by `wa-case-event-handler` `InitiationCaseEventHandler` when a CCD event matches a row in the initiation DMN. See [BPMN Workflows](../explanation/bpmn-workflows.md). |
| **CTSC** | Courts and Tribunals Service Centre. One of the five WA role categories. See [Overview](../explanation/overview.md). |
| **DateTypeConfigurator** | The `wa-task-management-api` component that processes date calculation key-value pairs from the Configuration DMN to produce final `dueDate`, `priorityDate`, and `nextHearingDate` values. See [DMN Task Configuration](../explanation/dmn-task-configuration.md). |
| **delayUntil** | A BPMN process variable (and optional Configuration DMN output) that postpones task activation until a specified datetime. When absent, the BPMN timer fires immediately via a past-date trick. See [BPMN Workflows](../explanation/bpmn-workflows.md). |
| **DMN** | Decision Model and Notation. Camunda decision tables used by service teams to configure WA task behaviour per jurisdiction. Seven DMN types exist per case type. See [DMN Task Configuration](../explanation/dmn-task-configuration.md) and [DMN Schema](dmn-schema.md). |
| **DmnValue** | The typed wrapper used in DMN evaluation requests and responses (`DmnValue<T>` with `value` and `type` fields). Factory methods include `dmnStringValue`, `dmnIntegerValue`, `booleanValue`, `jsonValue`. See [API: Workflow](api-workflow.md). |
| **dueDate** | One of three task dates tracked by WA. Mandatory; defaults to `now() + 2 days at 16:00`. Used for SLA reporting (late tasks). Not displayed in the UI directly. See [DMN Schema](dmn-schema.md). |
| **EvaluateDmnRequest** | The request body shape for the WA Workflow API's DMN evaluation endpoint. Contains a `variables` map of `DmnValue` entries. See [API: Workflow](api-workflow.md). |
| **EventInformation** | The Java class modelling the CCD case event message payload consumed by `wa-case-event-handler`. Fields: `EventInstanceId`, `CaseId`, `JurisdictionId`, `CaseTypeId`, `EventId`, `PreviousStateId`, `NewStateId`, `AdditionalData`. See [Case Event Handler](../explanation/case-event-handler.md). |
| **ExecutionType** | Enum defining how XUI launches and completes a task. Values: `MANUAL` (off-system), `CASE_EVENT` (requires a CCD event), `BUILT_IN` (app handles it). See [Task Lifecycle](../explanation/task-lifecycle.md). |
| **FEEL** | Friendly Enough Expression Language. The expression language used in Camunda DMN input/output expressions. WA DMNs use FEEL for null-safe navigation of `additionalData` fields and date calculations. See [DMN Task Configuration](../explanation/dmn-task-configuration.md). |
| **FIRST** | A Camunda DMN hit policy where only the first matching rule fires. Used in the allowed-days DMN. See [DMN Schema](dmn-schema.md). |
| **GIN index** | A PostgreSQL Generalized Inverted Index on the `tasks` table in `cft_task_db`. Only covers `ASSIGNED` and `UNASSIGNED` tasks with `indexed=true`. Used for production task search, controlled by LaunchDarkly flag `wa-task-search-gin-index`. See [Architecture](../explanation/architecture.md). |
| **idempotencyCheck** | A Camunda external service task topic. The `wa-workflow-api` worker for this topic prevents duplicate task creation by maintaining an `idempotent_keys` table. See [BPMN Workflows](../explanation/bpmn-workflows.md). |
| **idempotencyKey** | An MD5 hash of `eventInstanceId + taskId` generated by `wa-case-event-handler` and passed as a BPMN process variable. The `idempotencyCheck` worker uses it to detect duplicate task creation attempts. See [Case Event Handler](../explanation/case-event-handler.md). |
| **InitiationCaseEventHandler** | The `@Order(4)` handler in `wa-case-event-handler` that evaluates the initiation DMN and sends `createTaskMessage` to Camunda for each matching task. See [Case Event Handler](../explanation/case-event-handler.md). |
| **Initiation DMN** | The DMN table (`wa-task-initiation-{jurisdiction}-{caseType}`) that maps CCD event IDs and post-event states to task definitions. Hit policy: COLLECT. See [DMN Schema](dmn-schema.md). |
| **LEGAL_OPERATIONS** | One of the five WA role categories covering legal advisors and case officers. See [Overview](../explanation/overview.md). |
| **majorPriority** | A numeric task attribute used for coarse-grained task ordering. Default: `5000`. Lower values appear higher in lists. Set via the Configuration DMN. See [DMN Schema](dmn-schema.md). |
| **MARK_TO_RECONFIGURE** | A `POST /task/operation` operation type that sets matching tasks to `PENDING_RECONFIGURATION` state, triggering later re-evaluation of the Configuration DMN. See [API: Task Management](api-task-management.md). |
| **MessageReadinessConsumer** | The `wa-case-event-handler` background thread that promotes messages from `NEW` to `READY` state — but only when the ASB Dead Letter Queue is confirmed empty. See [Case Event Handler](../explanation/case-event-handler.md). |
| **minorPriority** | A numeric task attribute for fine-grained task ordering within the same `majorPriority` bucket. Default: `500`. See [DMN Schema](dmn-schema.md). |
| **nextHearingDate** | An optional task date surfaced in XUI. Allows prioritisation based on upcoming hearings. Configured via the Configuration DMN's date calculation engine. See [DMN Schema](dmn-schema.md). |
| **Permissions DMN** | The DMN table (`wa-task-permissions-{jurisdiction}-{caseType}`) that defines role-to-permission mappings for each task type. Hit policy: RULE ORDER. Produces `task_roles` rows. See [DMN Schema](dmn-schema.md). |
| **PermissionTypes** | The Java enum defining all 16 granular task permissions: `READ`, `REFER`, `OWN`, `MANAGE`, `EXECUTE`, `CANCEL`, `COMPLETE`, `COMPLETE_OWN`, `CANCEL_OWN`, `CLAIM`, `UNCLAIM`, `ASSIGN`, `UNASSIGN`, `UNCLAIM_ASSIGN`, `UNASSIGN_CLAIM`, `UNASSIGN_ASSIGN`. See [Access Control](../explanation/access-control.md). |
| **PENDING_RECONFIGURATION** | A task state (`PR`) indicating the task has been flagged by a `MARK_TO_RECONFIGURE` operation and is awaiting re-evaluation of its Configuration DMN attributes. See [Task States](task-states.md). |
| **priorityDate** | One of three task dates tracked by WA. Mandatory; defaults to `dueDate`. Used to order tasks in the Available Tasks list. See [DMN Schema](dmn-schema.md). |
| **processCategories** | A comma-separated string output from the initiation DMN (e.g. `"caseProgression,followUpOverdue"`). Each category becomes a `__processCategory__<cat>=true` BPMN process variable used to scope cancellation and warning messages. See [DMN Task Configuration](../explanation/dmn-task-configuration.md). |
| **ReconfigurationCaseEventHandler** | The `@Order(3)` handler in `wa-case-event-handler` that evaluates the cancellation DMN for `action=RECONFIGURE` rows and calls `POST /task/operation` on `wa-task-management-api`. See [Case Event Handler](../explanation/case-event-handler.md). |
| **RoleCategory** | Enum partitioning users and tasks into organisational groups: `JUDICIAL`, `LEGAL_OPERATIONS`, `ADMIN`, `CTSC`, `ENFORCEMENT`. See [Overview](../explanation/overview.md). |
| **RULE ORDER** | A Camunda DMN hit policy where all matching rules fire and their outputs are merged in row order (later rows override earlier rows for the same attribute key). Used in configuration and permissions DMNs. See [DMN Schema](dmn-schema.md). |
| **S2S** | Service-to-Service authentication. WA services use S2S tokens (`ServiceAuthorization` header) for all inter-service calls. No user bearer tokens are used on internal calls. See [Architecture](../explanation/architecture.md). |
| **SELECT FOR UPDATE SKIP LOCKED** | The PostgreSQL locking pattern used by `wa-case-event-handler`'s `DatabaseMessageConsumer` to pick up messages concurrently without contention. See [Case Event Handler](../explanation/case-event-handler.md). |
| **SendMessageRequest** | The request body shape for the WA Workflow API's message correlation endpoint (`POST /workflow/message`). Fields: `messageName`, `processVariables`, `correlationKeys`, `all`. See [API: Workflow](api-workflow.md). |
| **sensitive_task_event_logs** | A PostgreSQL table in `cft_task_db` (90-day TTL) that records access-control failures. Written asynchronously to avoid latency impact. Only accessible by SC-cleared personnel. See [Access Control](../explanation/access-control.md). |
| **task_roles** | The `cft_task_db` table storing per-task, per-role permission flags. One row per role per task, populated from the Permissions DMN during configuration. Used for all access-control decisions. See [Access Control](../explanation/access-control.md). |
| **task-supervisor** | The universal WA supervisor role. The Permissions DMN's first rule grants `Read,Manage,Cancel,Assign,Unassign,Complete` to this role for all task types. See [How-to: Write DMN Configuration](../how-to/write-dmn-configuration.md). |
| **TaskAutoAssignmentService** | The `wa-task-management-api` component that queries AM for `SPECIFIC` grant-type role assignments with `own=true` and `autoAssignable=true` to automatically assign tasks at creation time. See [Task Lifecycle](../explanation/task-lifecycle.md). |
| **TaskOperationType** | Enum of batch operations supported by `POST /task/operation`: `MARK_TO_RECONFIGURE`, `EXECUTE_RECONFIGURE`, `EXECUTE_RECONFIGURE_FAILURES`, `UPDATE_SEARCH_INDEX`, `CLEANUP_SENSITIVE_LOG_ENTRIES`, `PERFORM_REPLICATION_CHECK`. See [API: Task Management](api-task-management.md). |
| **TerminationProcess** | Enum recording how a task reached a terminal state: `EXUI_USER_COMPLETION`, `EXUI_CASE_EVENT_COMPLETION`, `EXUI_USER_CANCELLATION`, `EXUI_CASE_EVENT_CANCELLATION`. See [Task States](task-states.md). |
| **UNCONFIGURED** | The initial task state (`UCNF`) after Camunda creates the user task. The task-monitor INITIATION job picks up `UNCONFIGURED` tasks and calls `POST /task/{id}/initiation` to complete configuration. See [Task States](task-states.md). |
| **UNPROCESSABLE** | The terminal message state in `wa_case_event_messages_db` for messages that exhausted all retries (after 8 attempts) or received a non-retryable HTTP error (400/403/404). See [Case Event Handler](../explanation/case-event-handler.md). |
| **wa-case-event-handler** | Spring Boot service (port 8088) that consumes CCD case events from Azure Service Bus, persists them for deduplication and retry, and routes them through four handlers (Cancel, Warn, Reconfigure, Initiate). See [Case Event Handler](../explanation/case-event-handler.md). |
| **wa-message-cron-service** | Node.js Kubernetes CronJob that triggers `FIND_PROBLEM_MESSAGES` and related maintenance jobs on `wa-case-event-handler`. See [Architecture](../explanation/architecture.md). |
| **wa-standalone-task-bpmn** | The repository containing the single generic Camunda BPMN process (`wa-task-initiation-ia-asylum`) used for all jurisdictions. See [BPMN Workflows](../explanation/bpmn-workflows.md). |
| **wa-task-batch-service** | Node.js Kubernetes CronJob that triggers scheduled jobs (INITIATION, TERMINATION, RECONFIGURATION, etc.) on `wa-task-monitor`. See [Architecture](../explanation/architecture.md). |
| **wa-task-configuration-template** | The reference repository from which service teams copy and rename DMN files to create their jurisdiction-specific task configuration. See [How-to: Write DMN Configuration](../how-to/write-dmn-configuration.md). |
| **wa-task-management-api** | Spring Boot service (port 8087) providing the authoritative task store and access-control enforcement. Owns `cft_task_db`. See [API: Task Management](api-task-management.md). |
| **wa-task-monitor** | Spring Boot service (port 8077) that runs scheduled jobs to detect and recover stuck tasks (INITIATION, TERMINATION, RECONFIGURATION). Stateless — no database. See [Architecture](../explanation/architecture.md). |
| **wa-workflow-api** | Spring Boot service (port 8099) that proxies DMN evaluation and BPMN message correlation calls to the Camunda REST API. Also hosts the `idempotencyCheck` and `wa-warning-topic` external task workers. See [API: Workflow](api-workflow.md). |
| **wa_case_event_messages** | The primary table in `wa_case_event_messages_db` that stores CCD event messages with states `NEW`, `READY`, `PROCESSED`, `UNPROCESSABLE`. See [Case Event Handler](../explanation/case-event-handler.md). |
| **warnProcess** | The Camunda BPMN message name used to apply warning flags to running task processes without cancelling them. Sent by `wa-case-event-handler` `WarningCaseEventHandler`. See [BPMN Workflows](../explanation/bpmn-workflows.md). |
| **Work Allocation** | The HMCTS platform that turns CCD case events into actionable caseworker tasks. Comprises Case Allocation (long-term user-to-case links) and Task Management (individual units of work). See [Overview](../explanation/overview.md). |
| **workType** | A mandatory task attribute grouping tasks into logical work categories (e.g. `hearing_work`, `decision_making_work`, `routine_work`). Set in the Configuration DMN; used for filtering in XUI. See [Overview](../explanation/overview.md). |
| **XUI** | ExUI / `xui-webapp`. The HMCTS case management UI that surfaces WA tasks to caseworkers via the My Work and All Work screens. See [Overview](../explanation/overview.md). |
