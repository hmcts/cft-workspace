# XUI Glossary

Alphabetised definitions of terms used across the XUI documentation.

---

**`AbstractAppConfig`**
Angular DI token provided by each consuming app to supply all backend URLs to the `ccd-case-ui-toolkit` at runtime. Consuming apps implement the abstract class with concrete `getApiUrl()`, `getCaseDataUrl()`, etc. methods.
See: [Case UI Toolkit](../explanation/case-ui-toolkit.md)

**`applyProxy`**
The wrapper function in `api/lib/middleware/proxy.ts` that creates and mounts an `http-proxy-middleware` instance with auth injection, path rewriting, and error handling baked in.
See: [BFF Pattern](../explanation/bff-pattern.md), [Configure for New Service](../how-to/configure-for-new-service.md)

**`authInterceptor`**
Express middleware (`xuiNode.authenticate`) that validates a user session and injects `Authorization` and `ServiceAuthorization` headers before requests are forwarded to downstream services. Applied to every authenticated route and proxy.
See: [BFF Pattern](../explanation/bff-pattern.md)

**BFF (Backend-for-Frontend)**
The Express/Node.js server co-located with the Angular SPA in a single container. It handles OIDC session management, S2S token exchange, CSRF protection, and transparent proxying of browser requests to downstream HMCTS services. All three XUI apps share this pattern.
See: [BFF Pattern](../explanation/bff-pattern.md), [Architecture](../explanation/architecture.md)

**`ccd-case-ui-toolkit`**
The Angular component library (`@hmcts/ccd-case-ui-toolkit`) that renders all CCD case data in XUI â€” wizard forms, case views, workbasket filters, and the full field-type palette. Published independently to npm and pinned by version in each consuming app.
See: [Case UI Toolkit](../explanation/case-ui-toolkit.md), [Add a Component](../how-to/add-a-component.md), [Shared Libraries](shared-libraries.md)

**CDAM (Case Document Access Management)**
The `ccd-case-document-am-api` service that gates access to case documents. XUI proxies `/documentsv2` to CDAM v2, rewriting paths to `/cases/documents`. All document uploads and downloads from the browser go through CDAM, not the DM Store directly.
See: [Downstream Services](downstream-services.md), [Architecture](../explanation/architecture.md)

**`ComponentLauncher`**
A CCD field base type that delegates rendering to a named component registered in `PaletteService.componentLauncherRegistry`. Services specify the component via `#ARGUMENT(ComponentName)` in the `display_context_parameter`. Used by Case File View, Linked Cases, and Query Management.
See: [Case UI Toolkit](../explanation/case-ui-toolkit.md), [Add a Component](../how-to/add-a-component.md)

**CSP nonce**
A per-request `crypto.randomBytes(16)` value generated by the node-lib's `csp()` middleware. It is injected into `index.html` via string replacement of `{{cspNonce}}` and allows the Angular SPA's inline scripts to pass the Content Security Policy without using `unsafe-inline`.
See: [BFF Pattern](../explanation/bff-pattern.md), [Architecture](../explanation/architecture.md)

**ExUI / Expert UI**
The full name of the XUI product. Publicly branded as **MyHMCTS**. The product acronym is ExUI (pronounced "echs yoo eye"). Repository prefix is `rpx-xui-`; npm scope is `@hmcts/rpx-xui-*`.
See: [Overview](../explanation/overview.md)

**`FeatureToggleGuard`**
An Angular route guard from `@hmcts/rpx-xui-common-lib` that checks a route's `data.needsFeaturesEnabled` array using `getValueOnce`. Prevents navigation to a module when the corresponding LaunchDarkly flag is off; redirects to `data.featureDisabledRedirect`.
See: [Feature Flags](../explanation/feature-flags.md), [Add Feature Flag](../how-to/add-feature-flag.md)

**`FeatureToggleService`**
The abstract Angular service in `@hmcts/rpx-xui-common-lib` that provides `isEnabled()`, `getValue()`, `getValueOnce()`, and `getValueSync()` methods for flag consumption. The concrete implementation is `LaunchDarklyService`; the abstraction allows future replacement with Azure App Configuration.
See: [Feature Flags](../explanation/feature-flags.md), [Add Feature Flag](../how-to/add-feature-flag.md)

**`FeatureUser`**
The user context object sent to LaunchDarkly on login. Keyed by IDAM user ID, with `roles` (IDAM role list) and `orgId` fields. Role data enables LD segment-based targeting (e.g. judicial vs solicitors).
See: [Feature Flags](../explanation/feature-flags.md), [Add Feature Flag](../how-to/add-feature-flag.md)

**`getConfigValue<T>(ref)`**
The canonical BFF function (in `api/configuration/index.ts`) for type-safe config lookup. Takes a dotted-path reference constant from `api/configuration/references.ts` and calls `config.get(ref)`.
See: [Config Schema](config-schema.md), [BFF Pattern](../explanation/bff-pattern.md)

**LaunchDarkly (LD)**
The third-party feature flag service used for client-side, per-user dynamic flags in XUI. The LD client ID is a server-side Key Vault secret injected at bootstrap via `/external/config/ui`. The team has an active plan to migrate most flags away from LD toward static config and potentially Azure App Configuration.
See: [Feature Flags](../explanation/feature-flags.md), [Add Feature Flag](../how-to/add-feature-flag.md)

**`LaunchDarklyService`**
The concrete Angular implementation of `FeatureToggleService` in `@hmcts/rpx-xui-common-lib`. Wraps `launchdarkly-js-client-sdk`, emits default values before LD is ready, and subscribes to real-time flag changes via `client.on('change:<feature>')`.
See: [Feature Flags](../explanation/feature-flags.md), [Shared Libraries](shared-libraries.md)

**Manage Cases**
The primary caseworker and judicial interface (`rpx-xui-webapp`). Handles case viewing, event execution, task management (Work Allocation), hearings, role access, Notice of Change, global search, payments, and refunds. IDAM client ID: `xuiwebapp`.
See: [Overview](../explanation/overview.md), [Architecture](../explanation/architecture.md)

**Manage Organisations**
The solicitor-firm-facing application (`rpx-xui-manage-organisations`). Allows firms to manage their organisation, users, PBA accounts, and case access. IDAM client ID: `xuimowebapp`.
See: [Overview](../explanation/overview.md)

**`node-config`**
The `config` npm package used by all XUI BFFs. Loads `config/default.json` as the base, with env-var overrides declared in `config/custom-environment-variables.json`. `NODE_ENV` is always `production` in deployed environments; environment-specific overrides arrive via Helm.
See: [Config Schema](config-schema.md), [BFF Pattern](../explanation/bff-pattern.md)

**`@hmcts/rpx-xui-common-lib`**
Shared Angular library providing UI components (session dialog, cookie banner, loading spinner, GOV.UK wrappers), services (`TimeoutNotificationsService`, `LaunchDarklyService`, `FeatureToggleService`), guards, and directives. Consumed by all three XUI deployed apps.
See: [Shared Libraries](shared-libraries.md), [Session Management](../explanation/session-management.md)

**`@hmcts/rpx-xui-node-lib`**
Shared Node/Express library providing OIDC/OAuth2 authentication (via `openid-client` + Passport), S2S token exchange, Redis/file session management, CSP nonce generation, Helmet integration, and CSRF protection. The auth backbone for all three XUI BFFs.
See: [Shared Libraries](shared-libraries.md), [BFF Pattern](../explanation/bff-pattern.md), [Session Management](../explanation/session-management.md)

**`rpx-xui-translation`**
Angular library (`rpx-xui-translation`) providing the `rpxTranslate` pipe and `RpxTranslationService` for Welsh-language support. Fetches translations from `ts-translation-service` via debounced batch HTTP POST, caches results in IndexedDB (Dexie), and short-circuits to English when language is `'en'`.
See: [Translation](../explanation/translation.md), [Shared Libraries](shared-libraries.md)

**`PaletteService`**
The Angular service in `ccd-case-ui-toolkit` that dispatches CCD field rendering to the correct read/write component via a `switch` on `FieldTypeEnum`. Unknown field types return `UnsupportedFieldComponent` silently.
See: [Case UI Toolkit](../explanation/case-ui-toolkit.md), [Add a Component](../how-to/add-a-component.md)

**`ProxyConfig`**
The TypeScript interface in `api/lib/middleware/proxy.ts` that defines fields for an `applyProxy()` call: `source`, `target`, `rewrite`, `rewriteUrl`, `filter`, `middlewares`, `onReq`, `onRes`, and `ws`.
See: [BFF Pattern](../explanation/bff-pattern.md), [Configure for New Service](../how-to/configure-for-new-service.md)

**`RpxTranslatePipe`**
The Angular pipe (`rpxTranslate`) from `rpx-xui-translation` used in templates for Welsh translation. Declared as `pure: false` so it re-runs on change-detection when async results arrive. Must be imported through `RpxTranslationModule`.
See: [Translation](../explanation/translation.md)

**S2S (Service-to-Service)**
Authentication via TOTP tokens obtained from `rpe-service-auth-provider`. The BFF acquires S2S tokens using `@hmcts/rpx-xui-node-lib`'s `S2SAuth` middleware, which caches them in memory until their JWT `exp` claim expires. The XUI microservice name is `xui_webapp`.
See: [Session Management](../explanation/session-management.md), [Architecture](../explanation/architecture.md), [Shared Libraries](shared-libraries.md)

**`sessionTimeouts`**
An ordered array of `{ pattern, totalIdleTime, idleModalDisplayTime }` objects in `config/default.json`. The BFF matches user roles against each `pattern` (regex) and returns the first match. Values are in minutes; the Angular app converts to milliseconds. The catch-all pattern `"."` covers caseworkers with an 8-hour idle limit.
See: [Session Management](../explanation/session-management.md), [Config Schema](config-schema.md)

**`showFeature(feature)`**
BFF helper in `api/configuration/index.ts` that checks a boolean flag under `feature.*` in node-config. Takes a short key name (e.g. `'redisEnabled'`); prepends `'feature.'` internally.
See: [Config Schema](config-schema.md), [Feature Flags](../explanation/feature-flags.md)

**SRT (Service Regression Testing)**
The manual sign-off process for `ccd-case-ui-toolkit` releases that may affect service journeys. Service teams test their key journeys on a PR environment before the release is merged. High-risk toolkit changes require SRT; low-risk changes can go CI/CD.
See: [Add a Component](../how-to/add-a-component.md), [Shared Libraries](shared-libraries.md)

**Subtree proxy**
A prefix-based proxy mount (`app.use('/path', proxy)`) that forwards all requests under that path prefix to a downstream service, regardless of the specific path suffix or HTTP method. XUI uses this pattern for CCD API Gateway, documents, payments, and other transparent pass-through routes.
See: [BFF Pattern](../explanation/bff-pattern.md), [Architecture](../explanation/architecture.md)

**`TimeoutNotificationsService`**
Angular service from `@hmcts/rpx-xui-common-lib` that wraps `@ng-idle/core` and `@ng-idle/keepalive` to detect idle users and emit `sign-out`, `countdown`, and `keep-alive` events. Drives the session warning dialog.
See: [Session Management](../explanation/session-management.md), [Shared Libraries](shared-libraries.md)

**`ts-translation-service`**
The backend microservice that stores and serves Welsh phrase translations. Called by `rpx-xui-translation` via a BFF proxy at `/api/translation`. Translation management (CSV export/upload) is performed through `ccd-admin-web`.
See: [Translation](../explanation/translation.md)

**WA (Work Allocation)**
The task management platform (`wa-task-management-api`, `wa-workflow-api`) integrated into Manage Cases. XUI routes `/workallocation/*` through a local Express router (not a transparent proxy) that makes server-side Axios calls to WA APIs.
See: [Overview](../explanation/overview.md), [Downstream Services](downstream-services.md)

**`XuiNode`**
The singleton class from `@hmcts/rpx-xui-node-lib` that orchestrates session and auth middleware in a fixed order. BFFs call `xuiNode.configure(options)` to get an Express Router, and use `xuiNode.authenticate` as a per-route auth guard.
See: [Shared Libraries](shared-libraries.md), [Session Management](../explanation/session-management.md)

**`*xuilibFeatureToggle`**
Angular structural directive from `@hmcts/rpx-xui-common-lib` that subscribes to `FeatureToggleService.isEnabled()` reactively. The element appears and disappears in real time as the LaunchDarkly flag value changes. Preferred over `*exuiFeatureToggle` for LD-backed flags.
See: [Feature Flags](../explanation/feature-flags.md), [Add Feature Flag](../how-to/add-feature-flag.md)
